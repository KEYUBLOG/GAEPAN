# UI/UX 및 법적 신뢰도 개선 방안

> 목표: “법 지식 없이 대충 AI만 연결한 양산형 사이트” 느낌을 줄이고,  
> UI/UX와 법적 톤을 갖춘 **개판만의 법정 서비스**로 보이게 하는 방안 정리.

---

## 1. UI/UX 개선

### 1.1 시각 정체성 강화

| 현재 | 개선 방향 |
|------|------------|
| Geist + Arial, zinc/amber 일괄 적용 | **법정/판결소** 콘셉트에 맞는 톤 유지하면서 차별점 부여 |
| 로고가 "GAEPAN" 텍스트만 | 로고 옆 또는 푸터에 **짧은 슬로건** 고정 (예: "개인들의 판결소") |
| 다크 단일 테마 | 법원·선고 느낌을 주는 **포인트 컬러 사용 일관화**: 선고문 영역은 amber, 유죄/무죄는 red/blue로 이미 적용됨 → 이 구분을 섹션 제목·라벨에도 통일 |

**구체 작업**
- `globals.css` / layout: body에 적용된 `Arial` fallback을 유지하되, **선고문·판결문 블록**에만 `font-serif` 또는 법원 문서 느낌의 폰트(예: Noto Serif KR)를 적용해 “문서” 느낌 강화.
- 메인 CTA 영역("지금 기소하기") 주변에 **한 줄 설명** 추가:  
  "24시간 동안 배심원 투표 후, 선고문이 작성됩니다." → 흐름이 보이면 신뢰감 상승.

### 1.2 정보 계층 정리

| 문제 | 개선 |
|------|------|
| "사건 개요" / "AI 최종 선고" / "AI 상세 판결"이 비슷한 비중으로 나열 | **읽는 순서**를 명확히: ① 사건 개요 → ② 배심원 투표 결과 → ③ 최종 선고(유죄/무죄) → ④ 상세 선고문(근거). 섹션 번호 또는 작은 라벨("1. 사건", "2. 배심원 평결", "3. 선고", "4. 근거")로 단계 표시. |
| 판결 유보 시 "[ 판결 유보 : 판단 불가 ]"만 강조 | "배심원 투표가 동점이거나, 선고문 생성이 보류된 경우입니다." 같은 **한 줄 설명**을 바로 아래에 작게 표시. |

### 1.3 로딩·에러·빈 상태

- **선고문 생성 중**: "판결문 작성 중" 대신 **"선고문 초안 검토 중"**, "배심원 의견을 반영해 선고문을 작성하고 있습니다" 등 **법정 메타포**를 살린 문구로 통일.
- **에러 시**: "다시 시도해 주세요"와 함께 **"사건 접수가 지연되고 있습니다"** 등 서비스 톤에 맞는 메시지.
- **상세 판결 없음**: "상세 판결 근거가 기록되지 않은 사건입니다." 유지하되, 가능하면 **"이전 버전에서 작성된 사건이거나 기록이 누락되었을 수 있습니다"** 같이 이유를 한 줄 추가.

### 1.4 미세 인터랙션·접근성

- 유죄/무죄 선고 문구가 **처음 보일 때** 짧은 애니메이션은 이미 적용됨 → **스크린 리더**를 위해 `aria-live="polite"` 등으로 선고 결과 영역에 알림.
- "AI 대법관", "배심원" 등 **역할 라벨**에 `aria-label` 또는 툴팁으로 짧은 설명:  
  예) "최종 선고를 담당하는 개판 선고 시스템", "실제 이용자 투표 결과".

## 2. 법적 신뢰도·지식이 느껴지게

### 2.1 판례 연동을 “보이게” 하기

- **현재**: Judge API에서 `searchPrecedents()`로 law.go.kr 판례를 쓰고 있으나, **화면에는 노출되지 않음**.
- **개선**:  
  - 선고문(rationale)에 **"대법원 20XX…"** 형식 인용이 포함되면, 해당 문단을 **인용 블록** 스타일(들여쓰기, 작은 글씨, zinc-500)로 감싸서 "판례 인용"이 시각적으로 구분되게 표시.  
  - 또는 상세 판결 영역 위에 한 줄 배지:  
    **"이 선고는 유사 사건 판례(국가법령정보센터)를 참조해 작성되었습니다."**  
    (판례 검색 실패 시에는 노출하지 않거나, "참조 판례 없음"으로 표시해도 됨.)

### 2.2 용어·구조가 “법에 맞게” 보이게

- **사건 개요 / 선고 / 주문 / 상세 판결** 등 이미 쓰인 용어는 유지하되, **도움말 또는 소개 페이지**에서 짧게 정의:
  - 예: "선고: 재판 결과를 공식적으로 알리는 것.", "주문: 선고문에서 ‘무엇을 하라’고 정한 결론 부분."
- **About**에 다음을 추가:
  - "개판은 **기소·항변** 형식으로 사건을 접수합니다. 배심원(이용자) 투표 후 **선고문**이 작성되며, 가능한 경우 **대법원·하급심 판례**를 참조합니다."
  - "선고문은 **참고용**이며 **법적 효력이 없습니다.** 실제 법적 분쟁은 변호사·법원에 문의하세요."

### 2.3 참고용·법적 효력 없음 노출

- **About·이용약관**에만 두지 말고, **선고문이 보이는 모든 곳**에서 일관되게:
  - 선고 섹션 하단에 작은 글씨로:  
    **"본 선고는 참고용이며, 법적 효력이 없습니다."**
- (선택) 푸터에 한 줄:  
  "판례 검색: 국가법령정보센터(law.go.kr) 연동."

## 3. “AI 양산형” 느낌 줄이기

### 3.1 노출 문구에서 "AI" 과다 사용 완화

- **현재**: "AI 대법관", "AI 최종 선고", "AI 상세 판결", "AI 대법관 최종 확정" 등 반복.
- **개선**:
  - **"AI 대법관"** → **"개판 선고"** 또는 **"최종 선고"**로 바꾸고, 필요 시 툴팁/aria-label에서만 "자동 생성 선고문" 설명.
  - **"AI 최종 선고"** → **"최종 선고"**.
  - **"AI 상세 판결"** → **"선고문 (상세 근거)"** 또는 **"판결 근거"**.
  - 메타/OG 설명: "AI 대법관과 배심원들이 판결해드립니다" →  
    **"배심원 투표와 함께 최종 선고문을 만들어 드립니다"** 등 **사람 참여**를 앞에 두는 표현.

### 3.2 “사람 참여” 강조

- 배심원 수, 유죄/무죄 비율을 보여줄 때 **"실제 참여한 N명의 배심원 투표 결과"** 같은 문구를 가까이 배치.
- 공유 문구: "AI 대법관과 배심원의 판결을 확인하세요" →  
  **"개판 배심원 투표와 최종 선고를 확인하세요"**.

### 3.3 서비스만의 플로우 강조

- **실시간 재판소(전광판)**, **명예의 전당**, **기소 vs 항변** 선택 등은 이미 차별 요소임.
- 메인·About에 **한 줄씩** 명시:
  - "24시간 투표 후 선고문이 작성되는 **GAEPAN 법정**."
  - "**명예의 전당**에서 주차별 확정 사건을 볼 수 있습니다."

### 3.4 CTA·카피 톤

- "당신의 억울한 사연, AI 대법관이 판결해드립니다" →  
  **"사연을 접수하고, 배심원과 함께 결론을 냅니다"** 또는  
  **"기소장을 접수하면 24시간 뒤 선고문이 나옵니다."**
- "지금 기소하기 (공짜)" 유지해도 되되, 그 옆에 **"사건 접수"** 링크와 동등한 비중으로 두어 “법정” 느낌 유지.

## 4. 적용 우선순위 제안

| 우선순위 | 항목 | 예상 공수 | 효과 |
|----------|------|------------|------|
| 1 | 선고문 영역에 "참고용, 법적 효력 없음" 문구 항상 노출 | 낮음 | 법적 리스크·신뢰도 |
| 2 | "AI 대법관" → "최종 선고" 등 문구 정리 (메타·OG·화면) | 중간 | 양산형 느낌 감소 |
| 3 | 판례 참조 문구 추가 (선고문 위 배지 또는 About) | 낮음 | 법적 지식 인상 |
| 4 | 로딩/에러 메시지를 법정 메타포로 통일 | 낮음 | 톤 일관성 |
| 5 | 사건 개요 → 배심원 → 선고 → 근거 순서 라벨링 | 중간 | 정보 계층·가독성 |
| 6 | 선고문 블록에 serif/문서용 폰트 적용 | 낮음 | 시각 정체성 |
| 7 | About에 판례 연동·용어 설명 보강 | 낮음 | 신뢰·이해도 |

## 5. 수정 시 참고할 파일 위치

- **메타/OG/공유 문구**: `app/layout.tsx`, `app/trials/ongoing/layout.tsx`, `app/trials/completed/layout.tsx`, `app/page.tsx` (shareTitle 등), `app/rss/route.ts`, `app/rss.xml/route.ts`
- **선고문 UI 라벨**: `app/page.tsx`, `app/hall-of-fame/page.tsx`, `app/trials/ongoing/page.tsx`, `app/trials/completed/page.tsx` — "AI 대법관 선고", "AI 최종 선고", "AI 상세 판결" 검색 후 일괄 치환
- **참고용 문구 추가**: 위 네 페이지에서 선고 섹션 하단 또는 상단
- **판례 문구**: Judge API에서 `precedentBlock` 사용 여부를 클라이언트에 전달할 수 있다면, 해당 플래그로 "판례 참조됨" 배지 노출
- **About/소개**: `app/about/page.tsx`
- **전역 스타일·폰트**: `app/globals.css`, `app/layout.tsx`

이 순서대로 적용하면, 법적 톤과 UI/UX가 정리되면서 "대충 만든 AI 래퍼"가 아니라 **개판만의 법정 서비스**로 보이게 할 수 있다.
